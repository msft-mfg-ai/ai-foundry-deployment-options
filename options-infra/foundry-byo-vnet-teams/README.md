# Foundry BYO VNet + Teams-published agent

BYO VNet Foundry deployment that publishes an agent to **Microsoft Teams** via Azure Bot Service. Built from the shared modules plus:

- [`modules/bot/bot-service.bicep`](../modules/bot/bot-service.bicep) — Azure Bot Service backed by a UAMI, MsTeamsChannel enabled
- [`teams-app/`](./teams-app/) — Teams app icons (`color.png`, `outline.png`); the manifest is generated by Bicep and zipped by the postprovision hook

## Architecture

```
Teams client  ─►  Bot Framework channels  ─►  Foundry agent activityprotocol endpoint
                                     ▲
                                     │ JWT (aud = bot.msaAppId = UAMI clientId)
```

The bot service points directly at the Foundry agent's `…/protocols/activityprotocol` URL. The UAMI is granted `Azure AI User` on the Foundry account so it can reach the agent runtime.

## Deployment

```bash
azd up
```

The provision step deploys the infrastructure; the **postprovision** hook (in `azure.yaml`) then:

1. Runs `agents_v2/create_agents.py` to create the Foundry agent (`Teams Agent`).
2. Reads the `TEAMS_MANIFEST_JSON` Bicep output and writes it to `teams-app/build/manifest.json`.
3. Copies `teams-app/color.png` + `teams-app/outline.png` next to it.
4. Zips all three into `teams-app/build/teams-app.zip`.

Sideload `teams-app/build/teams-app.zip` into Teams (Teams Admin Center → **Manage apps** → **Upload**, or *Apps → Upload a custom app* for personal sideload).

## Customizing the manifest

The manifest is a Bicep object literal in [`main.bicep`](./main.bicep) (`teamsManifest`). Edit values like `developer.name`, `description.full`, or `permissions` there — the JSON is serialized via `string(...)` and emitted as `TEAMS_MANIFEST_JSON`.

Replace [`teams-app/color.png`](./teams-app/color.png) (192×192) and [`teams-app/outline.png`](./teams-app/outline.png) (32×32, transparent + white) with your own branding before publishing.

## Parameters

| Name | Default | Notes |
|---|---|---|
| `projectsCount` | `1` | Foundry projects to create |
| `apiServices` | `[]` | External MCP/OpenAPI services |

## Key outputs

| Output | Purpose |
|---|---|
| `BOT_APP_ID` | UAMI clientId (= bot `msaAppId`) — also the `botId` baked into the manifest |
| `TEAMS_APP_ID` | Deterministic GUID for the Teams app (stable across redeploys) |
| `TEAMS_MANIFEST_JSON` | Serialized manifest consumed by the postprovision hook |
| `botResourceId` / `botName` | Azure Bot Service identifiers |
| `AGENT_NAME` | Agent name configured on Foundry (`Teams Agent`) |
